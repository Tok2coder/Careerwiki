# HowTo 작성자 수익 모델 보고서

> 작성일: 2026-02-10
> 버전: v1.0 (초안)

---

## 1. 개요

커리어위키 HowTo 플랫폼의 지속 가능한 성장을 위해, **YouTube 파트너 프로그램(YPP)과 유사한 광고 수익 분배 모델**을 제안합니다. 양질의 콘텐츠를 작성하는 크리에이터에게 광고 수익의 일부를 분배하여, 콘텐츠 생태계의 선순환 구조를 구축하는 것이 목표입니다.

### 1.1 현재 상태

| 항목 | 현재 |
|------|------|
| 콘텐츠 작성 | 무료 (회원가입 필요) |
| 관리자 검수 | 수동 승인 (DRAFT → REVIEW → PUBLISHED) |
| 조회수 추적 | ✅ (`howtos.view_count` 컬럼) |
| 북마크 추적 | ✅ (`user_bookmarks` 테이블) |
| 광고 슬롯 | ✅ 출처 섹션과 댓글 섹션 사이에 배치 완료 |
| 작성자 수익화 | ❌ 없음 |

### 1.2 핵심 원칙

1. **작성자 이익 우선**: 양질의 콘텐츠가 플랫폼의 핵심 자산
2. **투명한 수익 분배**: 조회수 기반의 명확한 정산 기준
3. **품질 중심**: 조회수만이 아닌 콘텐츠 품질이 수익화 자격의 핵심
4. **단계적 도입**: 베타 → 정식 순차 확대

## 2. 수익화 자격 요건 (파트너 프로그램)

YouTube YPP를 참고하되, 커리어위키 규모에 맞게 조정한 자격 요건입니다.

### 2.1 파트너 등급

| 등급 | 요건 | 수익 분배율 | 비고 |
|------|------|-------------|------|
| **일반 작성자** | 가입 즉시 | 0% | 콘텐츠 작성 가능, 수익화 없음 |
| **루키 파트너** | 게시물 3개 이상 + 누적 조회수 1,000회 | 40% | 기본 광고 수익 분배 |
| **정규 파트너** | 게시물 5개 이상 + 누적 조회수 5,000회 | 55% | 프리미엄 광고 단가 적용 |
| **프로 파트너** | 게시물 10개 이상 + 누적 조회수 20,000회 | 65% | 최고 단가 + 우선 노출 혜택 |

### 2.2 자격 유지 조건

- **최소 활동**: 최근 6개월 내 1개 이상의 게시물 작성 또는 기존 게시물 업데이트
- **품질 기준 유지**: 블라인드 처리된 게시물이 전체의 10% 이하
- **커뮤니티 가이드라인 준수**: 신고 누적으로 인한 제재 기록 없음
- **미달 시**: 1개월 유예 기간 후 한 단계 강등

### 2.3 자격 박탈 사유

- 표절, 저작권 침해 확인 시 즉시 박탈
- 의도적인 조회수 어뷰징 (봇, 자동 새로고침 등) 확인 시
- 스팸성 저품질 콘텐츠 대량 생산 시
- 커뮤니티 가이드라인 중대 위반 시

## 3. 수익 구조

### 3.1 광고 수익원

| 광고 유형 | 위치 | 예상 CPM (₩) | 설명 |
|-----------|------|-------------|------|
| **디스플레이 광고** | 출처-댓글 사이 (`content-bottom` 슬롯) | 1,000~3,000 | 메인 광고 영역 |
| **네이티브 광고** | 사이드바 하단 (향후) | 2,000~5,000 | 관련 콘텐츠와 혼합 배치 |
| **스폰서 배너** | 히어로 섹션 하단 (향후) | 5,000~10,000 | 브랜드 파트너십 |

> **CPM** = Cost Per Mille (1,000회 노출당 비용)

### 3.2 수익 분배 계산 예시

**월간 시나리오**: 정규 파트너 (분배율 55%)가 작성한 HowTo 가이드

| 지표 | 수치 |
|------|------|
| 월간 조회수 | 10,000 |
| 평균 CPM | ₩2,000 |
| 총 광고 수익 | ₩20,000 |
| 작성자 몫 (55%) | **₩11,000** |
| 플랫폼 몫 (45%) | ₩9,000 |

**연간 시나리오**: 프로 파트너 (분배율 65%), 10개 게시물 보유

| 지표 | 수치 |
|------|------|
| 월간 총 조회수 (10개 합산) | 50,000 |
| 평균 CPM | ₩2,500 |
| 월간 총 광고 수익 | ₩125,000 |
| 작성자 월 수익 (65%) | **₩81,250** |
| 작성자 연 수익 | **약 ₩975,000** |

### 3.3 정산 규칙

- **정산 주기**: 월 1회 (매월 15일, 전월 실적 기준)
- **최소 출금 금액**: ₩10,000
- **미달 시**: 다음 달로 이월 (최대 12개월, 초과 시 소멸)
- **지급 방법**: 계좌이체 (본인 명의 확인 필수)
- **세금 처리**: 기타소득세 원천징수 후 지급

## 4. 기술 구현 계획

### 4.1 데이터베이스 스키마 확장

```sql
-- 파트너 프로그램 등록
CREATE TABLE howto_partners (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id TEXT NOT NULL REFERENCES users(id),
  tier TEXT NOT NULL DEFAULT 'rookie',  -- rookie | regular | pro
  status TEXT NOT NULL DEFAULT 'active', -- active | suspended | terminated
  applied_at TEXT NOT NULL DEFAULT (datetime('now')),
  approved_at TEXT,
  tier_updated_at TEXT,
  total_earnings INTEGER NOT NULL DEFAULT 0,  -- 누적 수익 (원)
  pending_balance INTEGER NOT NULL DEFAULT 0, -- 미정산 잔액 (원)
  UNIQUE(user_id)
);

-- 일별 조회수 기록 (정확한 수익 계산용)
CREATE TABLE howto_daily_views (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  howto_id TEXT NOT NULL REFERENCES howtos(id),
  view_date TEXT NOT NULL,  -- YYYY-MM-DD
  view_count INTEGER NOT NULL DEFAULT 0,
  ad_impressions INTEGER NOT NULL DEFAULT 0,
  estimated_revenue INTEGER NOT NULL DEFAULT 0,  -- 예상 수익 (원)
  UNIQUE(howto_id, view_date)
);

-- 월별 정산 내역
CREATE TABLE howto_payouts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  partner_id INTEGER NOT NULL REFERENCES howto_partners(id),
  period_month TEXT NOT NULL,    -- YYYY-MM
  total_views INTEGER NOT NULL,
  total_impressions INTEGER NOT NULL,
  gross_revenue INTEGER NOT NULL,   -- 총 광고 수익
  share_rate REAL NOT NULL,         -- 분배율 (0.40, 0.55, 0.65)
  author_revenue INTEGER NOT NULL,  -- 작성자 수익
  status TEXT NOT NULL DEFAULT 'calculated', -- calculated | approved | paid
  paid_at TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 조회수 어뷰징 감지 로그
CREATE TABLE howto_view_audit (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  howto_id TEXT NOT NULL,
  audit_date TEXT NOT NULL,
  suspicious_views INTEGER NOT NULL DEFAULT 0,
  legitimate_views INTEGER NOT NULL DEFAULT 0,
  flags TEXT,  -- JSON: 감지된 이상 패턴
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 4.2 어뷰징 방지 시스템

조회수 기반 수익 모델의 핵심은 **정확한 조회수 측정**입니다.

| 방지 기법 | 설명 |
|-----------|------|
| **IP 기반 중복 제거** | 동일 IP에서 24시간 내 중복 조회 1회로 산정 |
| **세션 기반 검증** | 최소 체류 시간 (10초) 미만 조회 제외 |
| **User-Agent 필터** | 봇/크롤러 트래픽 자동 제외 |
| **이상 패턴 감지** | 급격한 조회수 증가 시 자동 플래그 + 수동 검토 |
| **광고 노출 분리** | 실제 광고 impression만 수익 계산에 반영 |

### 4.3 작성자 대시보드 (신규 페이지)

파트너 작성자에게 제공할 수익 대시보드:

- **수익 요약**: 이번 달 예상 수익, 누적 수익, 미정산 잔액
- **게시물별 성과**: 각 HowTo의 조회수, 광고 노출, 수익 추이
- **등급 진행도**: 다음 등급까지 필요한 조회수/게시물 수
- **정산 내역**: 월별 정산 기록 및 지급 상태

- [x] 광고 슬롯 플레이스홀더 배치 (Job, Major, HowTo 상세 페이지)

### Phase 2: 베타 파트너 프로그램 (2~3개월)

- [ ] 파트너 DB 스키마 및 API 구현
- [ ] 파트너 신청/승인 관리 기능 (Admin 페이지)
- [ ] 작성자 대시보드 MVP
- [ ] 초대 기반 소수 베타 파트너 운영 (5~10명)
- [ ] 수익 분배 정산 프로세스 수동 진행

### Phase 3: 정식 오픈 (3~4개월)

- [ ] 자격 요건 충족 시 자동 파트너 승급
- [ ] 자동 월별 정산 시스템
- [ ] 정산 내역 자동 지급 (계좌이체 연동)
- [ ] 어뷰징 감지 고도화 (ML 기반 이상 패턴)

### Phase 4: 확장 (6개월~)

- [ ] 스폰서 콘텐츠 / 브랜드 파트너십 프로그램
- [ ] 사이드바 네이티브 광고 슬롯 추가
- [ ] 프리미엄 콘텐츠 유료 구독 모델 검토
- [ ] 작성자 협업 도구 (공동 집필, 시리즈 기능)

## 6. 리스크 분석

### 6.1 주요 리스크

| 리스크 | 영향도 | 발생 확률 | 대응 방안 |
|--------|--------|-----------|-----------|
| 조회수 어뷰징 | 높음 | 중간 | IP/세션 기반 필터 + 수동 검토 체계 |
| 저품질 콘텐츠 양산 | 높음 | 높음 | 관리자 검수 유지 + 품질 기준 명문화 |
| 광고 수익 불안정 | 중간 | 높음 | 최소 보장 기간 없음 명시 + 다양한 수익원 확보 |
| 세금/법률 이슈 | 높음 | 낮음 | 사업자 등록 + 세무 컨설팅 진행 |
| 작성자 이탈 | 중간 | 중간 | 등급별 혜택 차등화 + 커뮤니티 활성화 |

### 6.2 YouTube 모델과의 차이점

| 항목 | YouTube | 커리어위키 HowTo |
|------|---------|------------------|
| 콘텐츠 유형 | 동영상 | 텍스트 가이드 |
| 검수 방식 | 자동 (AI + 사후 검토) | **수동 관리자 사전 검수** |
| 수익화 기준 | 구독자 1,000 + 시청시간 4,000h | 게시물 3개 + 조회수 1,000 |
| 분배율 | 55% (고정) | 40~65% (등급별 차등) |
| 광고 유형 | 영상 전/중/후 광고 | 디스플레이 + 네이티브 |
| 결제 주기 | 월 1회 ($100 이상) | 월 1회 (₩10,000 이상) |

## 8. 결론

커리어위키의 **관리자 사전 검수 시스템**은 YouTube 대비 콘텐츠 품질 관리에 유리한 구조적 장점입니다. 이 강점을 활용하여:

1. **품질 보장된 콘텐츠**에 프리미엄 광고를 배치하고
2. **등급별 차등 분배율**로 장기 기여자를 우대하며
3. **투명한 정산 시스템**으로 작성자 신뢰를 확보하는

것이 핵심 전략입니다. 초기에는 소규모 베타로 시작하여 수익 모델을 검증한 후, 점진적으로 확대하는 것을 권장합니다.
