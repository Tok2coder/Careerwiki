# 직업 데이터 12개 필드 상세 스펙

## 개요
모든 필드는 `user_contributed_json` (TEXT 컬럼) 안에 JSON 문자열로 저장된다.
최소 JSON 크기: 2,000 bytes. 권장: 3,000~8,000 bytes.

---

## 1. way (string) — 진로 경로 ("되는 방법" 섹션)

**타입**: string (문자열, 절대 배열 금지!)
**위반 시**: formatRichText TypeError → HTTP 500
**렌더링 위치**: 페이지에서 **"되는 방법"** 섹션으로 표시됨

> ⚠️ `way` vs `detailReady` 혼동 금지:
> - `way` = 서술형 진로 경로 (자격요건·시험·진입경로를 **문장**으로 설명)
> - `detailReady` = 구조화된 준비 단계 (curriculum/recruit/training **배열** 목록)
> - 교육과정 목록을 way에 나열하지 마라 → detailReady에 넣어라

**규칙**:
- 최소 200자 이상, 권장 300~600자
- 인라인 참조번호 포함: `[1]`, `[2]` 형식
- 단계별 진로 경로를 구체적으로 서술 (학력 요건, 자격증, 채용 경로 포함)
- 마침표로 끝나는 완성된 문장 형식

**예시**:
```
"간호사가 되려면 4년제 간호학과 또는 3년제 간호전문대학을 졸업해야 한다.[1] 졸업 후 국가면허시험에 합격해야 하며,[2] 합격률은 약 95% 수준이다.[3] 취업은 대형 종합병원 공채, 중소병원 수시채용, 보건소·학교·산업체 등 다양한 경로가 있다.[4]"
```

**금지**:
```json
"way": ["단계1", "단계2", "단계3"]
```
배열로 저장 시 500 에러 즉시 발생.

## 2. overviewSalary (object) — 연봉 정보

**타입**: object

> ⚠️ **기존 데이터 변경 금지**: `wage` 값이 이미 존재하면 덮어쓰지 마라. `wage` + 구조화 포맷이 있어야 임금 바 차트가 렌더링됨. 이를 서술형으로 바꾸면 차트가 사라짐. 더 공신력 있는 출처 없으면 기존 값 유지.

**구조**:
```json
{
  "wage": "3,612",
  "wageSource": "캐릭터디자이너의 평균연봉(중앙값)은 3,612만원으로, 조사 대상 전체 직업 평균연봉(중앙값)인 4,072만원과 비교하여 낮은 수준입니다. (자료 : 워크넷, 직업명(2023))",
  "sal": "연봉 중위값은 약 3,612만 원이다.[1] 신입은 2,800~3,200만 원 수준이며, 10년차 이상 경력자는 5,000만 원 이상도 가능하다.[2]",
  "source": "직업백과, 워크넷(2024)"
}
```

**규칙**:
- `wage` **(필수 — 비주얼 렌더링에 사용)**: 연봉 중위값 숫자만 문자열로 (`","` 포함, 단위 없음, 예: `"3,612"`)
- `wageSource` **(권장)**: "직업명의 평균연봉(중앙값)은 X만원으로..." 포맷. 워크넷 등 공식 통계 자료 출처 명시
- `sal`: 구체적 수치(만 원 단위) + 인라인 참조번호. 중위값, 신입/경력 구간 언급 권장
- `source`: 출처 요약명 (선택적이지만 권장)
- **주의**: `wage` 없으면 임금 비주얼 차트가 렌더링되지 않음. 반드시 포함할 것
- 단순 "높다/낮다" 표현 금지, 수치 필수

## 3. overviewProspect (object) — 전망

**타입**: object
**구조**:
```json
{
  "main": "향후 5년간 고용은 보통 수준으로 유지될 전망이다.[1] 고령화로 인한 의료 수요 증가로 간호 인력 수요는 꾸준히 증가할 것으로 예측된다.[2]",
  "source": "직업백과, 커리어넷(2024)"
}
```

**규칙**:
- `main`: 5년 또는 10년 전망 + 인라인 참조번호
- 성장률 수치(%) 또는 고용 증감 방향 언급
- 기술 변화·사회 트렌드·정책 등 배경 원인 설명

**⚠️ 주의 — 면책 문구 자동 표시**:
- `overviewProspect.main`을 `user_contributed_json`에 넣으면 프론트엔드가 자동으로 "※ 위의 일자리 전망은 직업전문가들이..." 면책 문구를 표시한다
- 이 문구는 공식 워크넷·커리어넷 통계 기반 전망을 보호하기 위한 것이므로, user_contributed_json에 전망을 직접 기술할 때는 출처를 정확히 표기하여 면책 문구와 내용이 모순되지 않도록 해야 함
- 공식 데이터(merged_profile_json)가 이미 있는 직업은 overviewProspect 보완을 생략해도 무방

## 4. trivia (string) — 흥미로운 사실

**타입**: string
**규칙**:
- 최소 3개 이상의 흥미로운 사실을 하나의 문자열로 작성
- 각 사실은 줄바꿈(`\n`) 또는 번호(`1.`, `2.`) 구분
- 인라인 참조번호 포함
- 뻔한 직업 설명 금지. 의외의 사실, 역사, 재미있는 통계 위주

**예시**:
```
"1. 나이팅게일은 통계학자이기도 했다. 원형 차트를 최초로 사용해 병원 위생 개선의 필요성을 시각화했다.[1]\n2. 한국 간호사의 약 20%는 해외에서 근무 중이며, 미국·캐나다·독일에서 수요가 높다.[2]\n3. 남자 간호사 비율은 2024년 기준 약 9%로, 2010년 대비 3배 이상 증가했다.[3]"
```

## 5. detailWlb (object) — 워라밸 및 사회적 인식

**타입**: object
**구조**:
```json
{
  "wlb": "보통이하",
  "social": "높음",
  "wlbDetail": "3교대 근무로 야간·휴일 근무가 빈번하며, 간호사 1인당 담당 환자 수가 많아 체력적 부담이 크다.[1] 번아웃 비율이 높고 이직률도 타 의료직종 대비 높은 편이다.[2]",
  "socialDetail": "생명을 다루는 직업으로서 사회적 신뢰도와 인식이 높다.[3] 특히 팬데믹 이후 간호사에 대한 사회적 존경이 크게 높아졌다.[4]"
}
```

**wlb/social 등급 (5단계)**:
| 등급 | 의미 |
|------|------|
| 높음 | 매우 유연한 근무, 개인 시간 충분 |
| 보통이상 | 대체로 규칙적, 야근 드묾 |
| 보통 | 업계 평균 수준 |
| 보통이하 | 야근/교대 빈번, 스트레스 높음 |
| 낮음 | 극도의 불규칙 근무, 번아웃 위험 |

**규칙**:
- `wlb`, `social` 둘 다 위 5등급 중 하나로 정확히 입력
- `wlbDetail`, `socialDetail` 각각 최소 2문장 이상, 인라인 참조번호 포함

## 6. detailReady (object) — 준비 과정 ("교육과정/채용/훈련" 섹션)

**타입**: object
**구조**:
```json
{
  "curriculum": [
    "간호학과 4년 과정: 해부학, 생리학, 기초간호과학, 성인간호학, 아동간호학 등 이수",
    "임상실습 1,000시간 이상: 병원 각 과별 실습 필수",
    "영어 의학 용어 습득: 국제 논문·지침 활용 능력"
  ],
  "recruit": [
    "대형 종합병원 연 1~2회 공채: 서류→필기(인성/적성)→실기→면접",
    "중소병원 수시채용: 취업포털(사람인, 잡코리아) 활용",
    "보건소·학교보건: 지방직 공무원 시험 별도 응시"
  ],
  "training": [
    "전문간호사 자격취득: 응급, 중환자, 감염관리 등 13개 분야",
    "대학원 진학: 간호학 석·박사로 연구·교육직 진출",
    "해외 취업: 미국 NCLEX-RN, 독일 어학 자격 취득"
  ]
}
```

**규칙**:
- `curriculum`: 학습·교육 과정 (3~5개 항목)
- `recruit`: 채용 경로·전형 방식 (3~5개 항목)
- `training`: 심화·전문화 경로 (2~4개 항목)
- 각 항목: 구체적 이름·수치 포함. "열심히 공부한다" 같은 추상적 표현 금지

## 7. sidebarJobs (string[]) — 관련 직업

**타입**: string[] (문자열 배열)
**규칙**:
- DB `jobs` 테이블에 실존하는 직업명만 사용
- 사전 확인: `SELECT name FROM jobs WHERE name = '직업명';`
- 3~5개 (최대 10개까지 허용)
- 실제로 연관된 직업: 상위직, 하위직, 유사 직종, 협업 직종

**예시**:
```json
["조산사", "응급구조사", "물리치료사", "작업치료사", "임상병리사"]
```

**금지**: DB에 없는 직업명 사용 → 링크 깨짐

## 8. sidebarMajors (string[]) — 관련 전공

**타입**: string[] (문자열 배열)
**규칙**:
- DB `majors` 테이블에 실존하는 전공명만 사용
- 사전 확인: `SELECT name FROM majors WHERE name = '전공명';`
- 3~5개
- 직접 연관 전공 위주

**예시**:
```json
["간호학", "보건행정학", "의료경영학", "물리치료학", "임상병리학"]
```

## 9. sidebarCerts (string[]) — 관련 자격증

**타입**: string[] (문자열 배열)
**규칙**:
- 실제 존재하는 자격증·면허 이름
- 2~4개 (필수 자격증 → 우대 자격증 순서)
- 공식 명칭 사용 (약칭·별칭 금지)

**예시**:
```json
["간호사 면허", "전문간호사 자격증", "심폐소생술(CPR) 자격증", "응급처치 자격증"]
```

## 10. heroTags (string[]) — 직업 태그

**타입**: string[] (문자열 배열)
**규칙**:
- SEO 키워드 및 직업 특성 태그
- 4~6개 (최대 8개)
- 짧고 명확한 단어/구: "고소득", "전문직", "공무원", "의료", "자격증필수", "사람을돕는일"

**예시**:
```json
["의료", "전문직", "자격증필수", "사람을돕는일", "야간근무", "고용안정"]
```

## 11. youtubeLinks (object[]) — 유튜브 영상

**타입**: object[] (객체 배열)
**구조**:
```json
[
  {
    "url": "https://www.youtube.com/watch?v=XXXXXXXXXXX",
    "title": "간호사 현직자가 말하는 리얼 직업 이야기"
  }
]
```

> ⚠️ **`title` 필수**: title 없으면 URL 문자열이 제목으로 노출됨. oembed 응답 `title` 값을 사용하라.

**규칙**:
- 1~3개 (없으면 빈 배열 `[]`)
- oembed 검증 필수: WebFetch `https://www.youtube.com/oembed?url=VIDEO_URL&format=json`
  - 200 응답 + title 반환 = 정상 영상
  - 오류 응답 = 삭제/비공개 → 제외
- 최신(1~2년) 한국어 영상 우선
- **진로 탐색 목적 영상 우선**: "이 직업이 뭔지", "어떻게 되는지", "전망은 어떤지" 관점
  - 선호: 진로 가이드, 현직자 인터뷰, 전망 분석, 자격증 준비
  - 비추천: 단순 일과 vlog, 개인 일상
- 영상 제목은 oembed 응답의 `title` 값을 그대로 사용

## 12. _sources (object) — 출처 (가장 중요!)

**타입**: object
**A등급 포맷**:
```json
{
  "way": [
    {"id": 1, "text": "커리어넷 직업백과", "url": "https://www.career.go.kr/cnet/..."},
    {"id": 2, "text": "워크넷 직업정보", "url": "https://www.work.go.kr/..."}
  ],
  "overviewSalary.sal": [
    {"id": 3, "text": "직업백과", "url": "https://job.asamaru.net/..."}
  ],
  "overviewProspect.main": [
    {"id": 4, "text": "한국고용정보원", "url": "https://www.keis.or.kr/..."}
  ],
  "trivia": [
    {"id": 5, "text": "커리어넷 직업백과", "url": "https://www.career.go.kr/..."}
  ],
  "detailWlb.wlbDetail": [
    {"id": 6, "text": "직업백과", "url": "https://..."}
  ],
  "detailWlb.socialDetail": [
    {"id": 7, "text": "워크넷", "url": "https://..."}
  ]
}
```

**키 규칙**:
| 필드 | _sources 키 |
|------|-------------|
| way | `"way"` |
| overviewSalary.sal | `"overviewSalary.sal"` |
| overviewProspect.main | `"overviewProspect.main"` |
| trivia | `"trivia"` |
| detailWlb.wlbDetail | `"detailWlb.wlbDetail"` |
| detailWlb.socialDetail | `"detailWlb.socialDetail"` |
| detailReady | `"detailReady"` |

**id 규칙**:
- 전체 JSON 내 고유 순번 (1부터 연속, 중복 금지)
- 본문의 `[N]`은 해당 섹션 _sources 배열에서 `"id": N`인 항목을 참조

**금지 포맷**:
```json
"_sources": ["워크넷", "커리어넷"]
"_sources": {"way": "URL=커리어넷|URL2=워크넷"}
```

## 최소 품질 기준 요약

| 필드 | 최소 요건 |
|------|-----------|
| way | string, 200자+ |
| overviewSalary | object, sal에 수치 포함 |
| overviewProspect | object, main에 전망 기간 포함 |
| trivia | string, 3개 사실 이상 |
| detailWlb | object, 4개 키 모두 존재, 등급 정확 |
| detailReady | object, 3개 키 모두 존재, 각 3개+ 항목 |
| sidebarJobs | string[], 3개+, DB 실존 확인 |
| sidebarMajors | string[], 3개+, DB 실존 확인 |
| sidebarCerts | string[], 2개+ |
| heroTags | string[], 4개+ |
| youtubeLinks | object[], oembed 검증 (없으면 []) |
| _sources | object, A등급, URL 포함, id 연속 |
