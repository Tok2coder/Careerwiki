## 5. 핵심 기술 규칙 (반드시 준수)

### 이미지
- **R2 키**: `jobs/job-{slug}.webp` — `uploads/` prefix 절대 금지
- **DB image_url**: `/uploads/jobs/job-{slug}.webp?v={timestamp}`
- **프롬프트 스타일**: `src/services/geminiService.ts`의 JOB_PROMPT_TEMPLATE 참조
  - 비버: tiny teeth, big round glossy eyes, soft rounded cheeks, light-brown fluffy fur, creamy belly
  - 크기: 12% of the frame
  - 팔레트: deep navy, muted violet, soft teal glow
- **이미지 생성**: Evolink API (z-image-turbo 모델), 키는 `.dev.vars`에 있음
- **검증**: DB image_url 있어도 R2 파일 없을 수 있음 → 2단계 검증 필수

### 데이터
- **way 필드**: 반드시 string (배열로 저장 시 500 에러)
- **_sources**: `{ "url1": "사이트명 — 항목 설명", ... }` 형태 필수, URL만 넣기 금지
- **slug 규칙**: 공백 → 하이픈(-) 변환 (`slugifyName()`)
- **사이트 URL**: `careerwiki.org/job/슬러그` (.kr/jobs 절대 금지)
- **기존 데이터 변경 금지**: 임금 등 기존 데이터 수정 시 더 나은 출처 필수
- **heroCategory 형식**: `{"type":"breadcrumb","large":"대분류","medium":"중분류"}`
- **10대 분류**: 경영·사무·금융·보험직 / IT·연구직 및 공학 기술직 / 교육·법률·사회복지·경찰·소방직 및 군인 / 보건·의료직 / 예술·디자인·방송·스포츠직 / 미용·여행·숙박·음식·경비·청소직 / 영업·판매·운전·운송직 / 설치·정비·생산직 / 건설·채굴직 / 농림어업직

### 배포
- **worktree에서 배포 금지** — main에서 배포하지 않으면 다음 main 배포 시 롤백됨
- 반드시 main 브랜치에서 작업+배포

### 중복 체크
- 직업/howto 생성 전 DB에서 유사 항목 검색 필수
- YouTube 링크는 oembed API로 실존 검증 + 최신(1~2년 이내) 우선
