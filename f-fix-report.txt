[1] aiAnalysisService.ts — lastRowId → meta.last_row_id (2곳)
─────────────────────────────────────────────────────────────
Before:
  const requestId = Number(result.lastRowId)
  const resultId  = Number(insert.lastRowId)

After:
  const requestId = Number(result.meta.last_row_id)
  const resultId  = Number(insert.meta.last_row_id)

동작 차이:
  lastRowId는 구 D1 SDK API로 현재 D1Result 타입에 존재하지 않음.
  런타임에 undefined → Number(undefined) = NaN.
  이후 "SELECT id = NaN" 쿼리가 행을 못 찾아
  "failed to create analysis request" 예외 발생, 분석 저장 전체 실패.
  수정 후: 올바른 INSERT row id로 후속 SELECT 성공, 정상 저장.

[2] vis-synthesizer.ts — callOpenAI 호출 시그니처 수정
─────────────────────────────────────────────────────
Before:
  const response = await callOpenAI(openaiApiKey, {
    model: OPENAI_MODEL,
    messages: [
      { role: 'system', content: systemPrompt },
      { role: 'user', content: userPrompt },
    ],
    temperature: 0.3,
    max_tokens: 200,
  })
  const content = response.choices[0]?.message?.content || '{}'

After:
  const response = await callOpenAI(openaiApiKey, [
    { role: 'system', content: systemPrompt },
    { role: 'user', content: userPrompt },
  ], {
    model: OPENAI_MODEL,
    temperature: 0.3,
    max_tokens: 200,
  })
  const content = response.response || '{}'

동작 차이:
  callOpenAI의 실제 시그니처: (key, messages[], options) → { response: string }
  구 형태로 호출 시: options 인자 위치에 객체 통째로 들어가 messages 필드가 무시됨.
  + choices 없어서 response.choices[0] = undefined → content 항상 '{}'.
  vis-synthesizer의 혼란 포인트 분석(tension 분석, summary/suggested_angle 생성)이
  항상 fallback 값만 반환하던 버그 수정.
  수정 후: 실제 OpenAI 응답 파싱, 혼란 분석 정상 작동.
